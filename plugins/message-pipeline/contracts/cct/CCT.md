# Compact Conversation Transcript (CCT)

## 버전

- `CCT2`: 분 단위 정확 시각 행. `exact` 프로필의 기본 형식.
- `CCT3`: 비활동 간격 기반 세션 행. `schedule` 프로필의 기본 형식.

인코딩은 UTF-8, 줄 끝은 LF다. 첫 줄은 반드시 버전과 설정을 선언한다.

```text
!CCT3|g=30|z=Asia/Seoul
T|K001
D|260722
S|0930
A|내일 10시 가능해?
B|Y
|장소는 전과 같아
```

## 레코드

| Prefix | CCT2 | CCT3 | 의미 |
|---|---|---|---|
| `!` | 필수 | 필수 | 버전과 설정 (`g`: 세션 분, `z`: 출력 시간대) |
| `T` | 필수 | 필수 | 모델용 스레드 별칭 |
| `D` | 필수 | 필수 | `YYMMDD` 현지 날짜 |
| `S` | 없음 | 세션마다 | `HHmm` 세션 첫 메시지 시각 |
| 그 외 | `HHmm|speaker|text` | `speaker|text` | 메시지 행 |

CCT2 메시지 행에서 빈 `HHmm`은 직전 분을, 빈 `speaker`는 직전 화자를 상속한다. CCT3 메시지 행에서 빈 `speaker`는 직전 화자를 상속한다. `T`, `D`, `S`가 바뀌면 해당 하위 상속 상태를 초기화한다.

화자 `A`는 자기 자신이다. 다른 화자는 스레드 안에서 `B`, `C`, ..., `Z`, `AA` 순으로 할당한다. `Y`, `N`, `?` 본문은 각각 확인/긍정, 거절/부정, 질문 반응의 축약이다.

## 이스케이프

필드에서 다음 문자만 역슬래시로 이스케이프한다.

| 원문 | 인코딩 |
|---|---|
| `\\` | `\\\\` |
| `|` | `\\|` |
| LF | `\\n` |
| CR | `\\r` |

파서는 알 수 없는 `\\x`를 거부해야 한다. 빈 본문은 허용하지 않는다.

## 정렬과 범위

스레드는 별칭 오름차순, 메시지는 `(timestamp_utc, source_message_id)` 오름차순이다. 범위는 시작 포함·종료 제외다. 날짜가 바뀌면 새 `D`가 필요하고 CCT3 세션도 새로 시작한다.

## 개인정보 경계

CCT에는 표시 이름, 연락처 핸들, 원본 chat/message/author ID를 넣지 않는다. 그 매핑과 정확한 UTC 시각은 소유자 전용 중앙 상태에만 둔다.
