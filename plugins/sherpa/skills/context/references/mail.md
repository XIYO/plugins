# 메일 컨텍스트 소스

메일은 `Context > MailMessage`로 분류한다. Gmail·iCloud Mail·Outlook 같은 제공자와 계정은 소스 속성이고, 메일 자체가 제공자와 같은 층위는 아니다.

## 수집

- 연결된 메일 앱이 있으면 해당 앱의 검색·읽기 도구를 사용한다.
- 날짜, 발신자, 수신자, 제목, 라벨 중 요청에 필요한 조건만 사용한다.
- 전체 받은편지함을 읽지 않는다.
- 같은 스레드의 이전 본문을 이미 알고 있다면 새 메시지와 필요한 인용 범위만 읽는다.
- 첨부는 사용자가 요청하지 않으면 메타데이터만 다룬다.

## 정규화

분석할 때 다음 공통 의미만 유지한다.

- 제공자와 계정의 불투명 식별자
- 스레드와 메시지의 안정 식별자
- 발신·수신 시각
- 발신자·수신자 역할
- 제목과 필요한 본문
- 첨부 메타데이터

메일 주소와 실명은 최종 답변에 필요할 때만 표시한다. 로그와 테스트 fixture에는 넣지 않는다.

## 가용성

연결된 메일 앱이 없으면 브라우저 로그인으로 우회하지 않는다. 해당 소스를 `사용 불가`로 명시하고 다른 Context 소스의 결과만 계속 처리한다.
