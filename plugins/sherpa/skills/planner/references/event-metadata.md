# 이벤트 메타데이터 계약

## 문서 문법

```text
<요약 1> · <요약 2> · <요약 3>

<구역>
• <필드>: <값>

@schema: <스키마 이름>@<MAJOR>[.<MINOR>]
```

- 요약은 정확히 세 항목이며 구분자는 공백을 포함한 ` · `다.
- 구역명은 단독 행이다. 필드는 `• `로 시작하고 필드명과 값은 `: `로 구분한다.
- 빈 구역, 빈 필드명·값, 중복 구역·필드, 등록되지 않은 구역·필드는 허용하지 않는다.
- 스키마 선언은 비어 있지 않은 마지막 행에 한 번만 둔다.
- 줄바꿈은 입력에서 LF와 CRLF를 허용하고 출력은 LF로 정규화한다.

## 버전

메타데이터 계약은 `MAJOR.MINOR`만 사용한다. `@1`은 `@1.0`의 정규 표기이며, `@1.1`처럼 0이 아닌 MINOR만 표시한다.

- MAJOR: 필수 구역·필드 추가, 이름 변경·삭제, 필드 의미나 형식 변경, 문법 변경, 기존 유효 문서를 거부하는 검증 강화.
- MINOR: 선택 구역·필드 추가처럼 기존 문서와 기존 의미를 그대로 유지하는 확장.
- PATCH: 사용하지 않는다. 일정 메모는 수정 이력을 보존하지 않고, 계약의 오탈자 수정은 데이터 해석에 영향을 주지 않기 때문이다.

플러그인과 실행 파일의 배포 버전은 이 계약과 별개로 일반 SemVer `MAJOR.MINOR.PATCH`를 사용한다.

## `xiyo.calendar.telecom-billing@1`

요약 의미: `<일정 의미> · <서비스 제공자> · <결제수단 식별자>`

| 구역 | 필드 | 필수 | 의미 |
|---|---|---:|---|
| 서비스 | 제공자 | 예 | 통신사 또는 인터넷 사업자 |
| 서비스 | 상품 | 예 | 요금제 또는 인터넷 상품 |
| 서비스 | 종류 | 아니요 | 휴대전화, 인터넷 등 |
| 서비스 | 식별자 | 아니요 | 마스킹한 회선 등 |
| 서비스 | 속도 | 아니요 | 인터넷 상품 속도 |
| 계약 | 시작일 | 아니요 | 서비스 시작일 |
| 계약 | 종료일 | 아니요 | 계약 종료일 |
| 계약 | 할인기간 | 아니요 | 할인 적용 범위 |
| 계약 | 약정기간 | 아니요 | 약정 기간 또는 범위 |
| 계약 | 결합 | 아니요 | 결합상품 이름 |
| 납부 | 청구일 | 예 | 매월 청구 또는 출금 기준일 |
| 납부 | 결제수단 | 예 | 마스킹한 카드·계좌 식별자 |
| 납부 | 청구서 | 아니요 | 청구서 수신 주소 |
| 계정 | 로그인 | 아니요 | 로그인에 사용하는 연결 계정 |

## `xiyo.calendar.card-payment@1`

요약 의미: `<일정 의미> · <카드 상품> · <발급사 끝번호>`

| 구역 | 필드 | 필수 | 의미 |
|---|---|---:|---|
| 카드 | 발급사 | 예 | 카드 발급사 |
| 카드 | 상품 | 예 | 카드 상품명 |
| 카드 | 끝번호 | 예 | 마스킹한 카드 끝번호 |
| 카드 | 유효기간 | 아니요 | 카드 유효기간 |
| 결제 | 결제일 | 예 | 카드 대금 결제일 |
| 결제 | 결제계좌 | 아니요 | 마스킹한 결제 계좌 |
| 명세서 | 수신처 | 아니요 | 명세서 수신 주소 |

## 명령

```bash
sherpa planner metadata validate note.txt
sherpa planner metadata validate note.txt --json
sherpa planner metadata parse note.txt
sherpa planner metadata spec

sherpa planner metadata render \
  --schema xiyo.calendar.telecom-billing@1 \
  --summary '요금 청구일 · 예시모바일 · 예시카드 1234' \
  --field '서비스.제공자=예시모바일' \
  --field '서비스.상품=데이터 11GB' \
  --field '납부.청구일=매월 9일' \
  --field '납부.결제수단=예시카드 · 끝 1234'
```

`render`는 스키마에 선언된 구역·필드 순서로 정규화한다. `parse`는 문법만, `validate`는 문법과 스키마를 함께 검사한다.
